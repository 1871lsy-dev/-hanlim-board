# 리팩토링 계획: 한림바이오팜 업무 현황 보드

## 목표
`index.html` 단일 파일(502줄)을 역할별로 분리해 유지보수성 향상

**제약:** Firebase v8 CDN 유지, 빌드 도구 없음, Vercel 정적 배포 유지

---

## 최종 파일 구조

```
/
├── index.html          (HTML 뼈대 + script/link 태그만)
├── style.css           (전체 CSS)
└── js/
    ├── firebase-config.js  (Firebase 초기화, db 전역 노출)
    ├── state.js            (상수, DEFAULT, 유틸 함수, 전역 S, fbSave/fbRemove)
    ├── components.js       (el, av, pill, modal, asel, tcard, ticon, weekCal, re/cm/chkAnim)
    ├── tabs.js             (tabHome ~ tabJournal)
    ├── modals.js           (mSettings, mDetail, mAdd*, mViewJournal)
    └── app.js              (render, Firebase 리스너, 초기 render() 호출)
```

## 구현 순서

1. `style.css` 생성 — `<style>` 내용 복사
2. `js/firebase-config.js` 생성
3. `js/state.js` 생성
4. `js/components.js` 생성
5. `js/tabs.js` 생성
6. `js/modals.js` 생성
7. `js/app.js` 생성
8. `index.html` 정리 (인라인 `<script>/<style>` 제거, 외부 파일 참조로 교체)
9. `git add . && git commit -m "파일 분리 리팩토링" && git push`

## 주의 사항

- **const 중복 선언 금지:** 동일 변수명을 여러 파일에 선언하면 오류
- **로드 순서 엄수:** 위 순서대로 script 태그 배치
- **Firebase 경로:** `hanlim/` prefix 그대로 유지 (변경 금지)
- **전방참조 안전:** `re()` → `render()` 호출은 로드 완료 후 실행되므로 문제없음
